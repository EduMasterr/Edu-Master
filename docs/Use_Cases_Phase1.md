# حالات الاستخدام (Use Cases) - EduMaster Pro
**الوثيقة:** تفاصيل حالات الاستخدام لكل دور
**المرحلة:** 1 (التخطيط والتحليل)

---

## 1. حالات إدارة النظام (Super Admin Use Cases)

### UC-SA-01: تسجيل الدخول (Login)
- **الممثل:** Super Admin.
- **الوصف:** الوصول إلى لوحة تحكم النظام العامة.
- **الخطوات:**
  1. إدخال اسم المستخدم وكلمة المرور.
  2. النظام يتحقق من البيانات (Authentication).
  3. توجيه إلى لوحة التحكم الرئيسية (Super Admin Dashboard).

### UC-SA-02: إدارة الفروع (Manage Branches)
- **الممثل:** Super Admin.
- **الوصف:** إضافة، تعديل، حذف، أو عرض بيانات الفروع.
- **الخطوات:**
  1. عرض قائمة الفروع الحالية.
  2. النقر على "إضافة فرع".
  3. إدخال (اسم الفرع، المدير، العنوان، الهاتف، البريد).
  4. حفظ البيانات.
  5. تعيين مدير فرع (Create Branch Admin Account).

### UC-SA-03: إدارة الصلاحيات (Manage Roles & Permissions)
- **الممثل:** Super Admin.
- **الوصف:** تحديد الصلاحيات لكل دور (Admin, Teacher, Employee).
- **الخطوات:**
  1. الوصول لشاشة "الصلاحيات والأدوار".
  2. إنشاء دور جديد (مثل: "مسؤول مبيعات الفرع 1").
  3. تحديد الصلاحيات المسموحة (قراءة، كتابة، حذف).
  4. تعيين الدور لمستخدم محدد.

## 2. حالات مدير الفرع (Branch Admin Use Cases)

### UC-BA-01: متابعة أداء الفرع (Branch Dashboard Overview)
- **الممثل:** Branch Admin.
- **الوصف:** عرض مؤشرات الأداء الحالية للفرع فقط.
- **المعلومات:** عدد الطلاب النشطين، الحضور اليومي، الإيرادات اليومية، المجموعات المفتوحة.

### UC-BA-02: إدارة الموظفين والمدرسين (Employee/Teacher Management)
- **الممثل:** Branch Admin.
- **الوصف:** إضافة موظفين/مدرسين جدد داخل الفرع.
- **الخطوات:**
  1. النقر على "الموظفين".
  2. إضافة موظف جديد وتحديد دوره (استقبال/حسابات/مدرس).
  3. تعيين جدول مواعيد العمل.

### UC-BA-03: إدارة الكورسات والمجموعات (Course/Group Planning)
- **الممثل:** Branch Admin.
- **الوصف:** إنشاء مجموعات دراسية جديدة وتعيين المدرسين.
- **الخطوات:**
  1. اختيار الكورس (مثل: لغة إنجليزية مستوى 1).
  2. إنشاء مجموعة جديدة (Group A).
  3. تحديد المواعيد (سبت/اثنين/أربعاء 4-6).
  4. تعيين المدرس والقاعة.
  5. تحديد الحد الأقصى للطلاب.

## 3. حالات الموظف (Employee Use Cases - Reception/Sales)

### UC-EMP-01: تسجيل طالب جديد (Student Registration)
- **الممثل:** Receptionist / Sales.
- **الوصف:** إدخال بيانات طالب جديد في النظام.
- **الخطوات:**
  1. النقر على "تسجيل طالب".
  2. إدخال البيانات الشخصية (الاسم، الهاتف، ولي الأمر، المدرسة، الصف).
  3. رفع صور المستندات (إن وجدت).
  4. حفظ البيانات وإنشاء ملف الطالب.

### UC-EMP-02: تحصيل الرسوم (Payment Collection)
- **الممثل:** Accountant / Receptionist.
- **الوصف:** استلام دفعة مالية من طالب.
- **الخطوات:**
  1. البحث عن الطالب بالاسم أو الكود.
  2. عرض المستحقات المالية.
  3. إدخال المبلغ المدفوع ونوع العملية (نقدي/فيزا).
  4. إصدار إيصال دفع (طباعة/PDF).
  5. تحديث رصيد الطالب تلقائياً.

### UC-EMP-03: الاستعلامات (Inquiries Management)
- **الممثل:** Customer Service.
- **الوصف:** الرد على استفسارات العملاء وتسجيل اهتماماتهم (Leads).
- **الخطوات:**
  1. تسجيل بيانات المتصل (اسم، هاتف، اهتمام).
  2. تحديد حالة المتابعة (Follow-up Date).

## 4. حالات المدرس (Teacher Use Cases)

### UC-TCH-01: تسجيل الحضور (Take Attendance)
- **الممثل:** Teacher.
- **الوصف:** تسجيل حضور وغياب الطلاب في الحصة.
- **الخطوات:**
  1. الدخول لحسابه واختيار المجموعة الحالية.
  2. عرض قائمة الطلاب.
  3. تحديد الحالة (حاضر/غائب/تأخير) لكل طالب.
  4. حفظ السجل (يتم إرسال تنبيهات تلقائية للغائبين - Placeholder).

### UC-TCH-02: إدخال الدرجات والتقييم (Grading)
- **الممثل:** Teacher.
- **الوصف:** رصد درجات الاختبارات والواجبات.
- **الخطوات:**
  1. اختيار نوع التقييم (Quiz 1, Midterm).
  2. إدخال الدرجة لكل طالب.
  3. إضافة ملاحظات سلوكية (إن وجدت).

## 5. حالات الطالب (Student Use Cases)

### UC-STD-01: عرض الجدول الدراسي (View Schedule)
- **الممثل:** Student.
- **الوصف:** معرفة مواعيد الحصص القادمة.
- **الشاشة:** تقويم أسبوعي يوضح المادة، الوقت، والقاعة/الرابط.

### UC-STD-02: متابعة الدرجات والمالية (View Grades & Finance)
- **الممثل:** Student.
- **الوصف:** الاطلاع على سجل الدرجات والدفعات.
- **المعلومات:** قائمة الدرجات، المدفوعات السابقة، والمبالغ المستحقة.

## 6. حالات ولي الأمر (Parent Use Cases)

### UC-PAR-01: متابعة الأبناء (Monitor Children)
- **الممثل:** Parent.
- **الوصف:** لوحة موحدة لمتابعة جميع الأبناء المسجلين.
- **المعلومات:** تقارير الحضور، الدرجات، والمطالبات المالية لكل ابن.
